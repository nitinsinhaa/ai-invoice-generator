# Deploy AI Invoice Generator

Your code is on GitHub: **https://github.com/nitinsinhaa/ai-invoice-generator**

Recommended path: **Render** (free tier, connects to GitHub, includes PostgreSQL).

---

## Option 1 — Render (recommended)

### Step 1: Create a Render account

Sign up at [render.com](https://render.com) and connect your GitHub account.

### Step 2: Deploy with Blueprint

1. Open [Render Blueprints](https://dashboard.render.com/blueprints)
2. Click **New Blueprint Instance**
3. Select repo **`nitinsinhaa/ai-invoice-generator`**
4. Render reads `render.yaml` and creates:
   - **PostgreSQL** database (`ai-invoice-db`)
   - **API** web service (`ai-invoice-api`)
   - **Frontend** static site (`ai-invoice-app`)
5. When prompted, set **`GEMINI_API_KEY`** (your Google AI Studio key)
6. Click **Apply**

First deploy takes ~5–10 minutes. Migrations run automatically on API startup (`npm run migrate` in start command).

### Step 3: Your live URLs

After deploy, open your **API service URL** in the browser — it serves both the React app and the API:

| What | URL |
|------|-----|
| **App (open this)** | `https://ai-invoice-api-xxxx.onrender.com` |
| Health check | `https://ai-invoice-api-xxxx.onrender.com/api/health` |

Render adds a suffix to the URL (e.g. `ai-invoice-api-5bah.onrender.com`). Find yours under **ai-invoice-api** → **Settings**.

> **Note:** One service hosts the full app (frontend + API). You can ignore or delete the old **ai-invoice-app** static site if it was created by an earlier blueprint.

### Step 4: Seed demo data (optional)

In Render → **ai-invoice-api** → **Shell**:

```bash
npm run seed
```

Demo login: `demo@aiinvoice.com` / `demo123456`

### Step 5: Email (optional)

In **ai-invoice-api** → **Environment**, add:

```
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your@gmail.com
EMAIL_PASSWORD=your_app_password
EMAIL_FROM=AI Invoice <your@gmail.com>
```

## Render notes

- **Free tier** services spin down after ~15 min idle; first request may take 30–60s.
- **JWT_SECRET** is auto-generated by Render.
- **CORS** uses `FRONTEND_URL` linked from the frontend service automatically.
- **Never** commit `.env` — set secrets in Render dashboard only.

## Option 2 — Docker Compose (VPS / local server)

Requires Docker installed on the server.

```bash
# On the server
git clone https://github.com/nitinsinhaa/ai-invoice-generator.git
cd ai-invoice-generator

# Create env file (do not commit)
cat > .env << 'EOF'
GEMINI_API_KEY=your_key
JWT_SECRET=change_to_long_random_secret_min_32_chars
FRONTEND_URL=https://your-domain.com
EOF

# Edit docker-compose.yml: set FRONTEND_URL and VITE_API_URL build arg to your domain
docker compose up --build -d
```

Point your domain at the server; put **nginx/Caddy** in front for HTTPS.

## Option 3 — Manual (Node on a VPS)

```bash
# PostgreSQL on the server, then:
cd backend
cp .env.example .env   # fill DATABASE_URL or DB_* + JWT_SECRET + FRONTEND_URL
npm ci --omit=dev
npm run migrate
NODE_ENV=production npm start

# Frontend
cd ../frontend
echo "VITE_API_URL=https://api.yourdomain.com/api" > .env
npm ci && npm run build
# Serve dist/ with nginx
```


## Troubleshooting

| Issue | Fix |
|-------|-----|
| CORS error in browser | Use one URL: `https://ai-invoice-api-xxxx.onrender.com` for the app, or redeploy API after latest CORS fix (allows `*.onrender.com`). Set `FRONTEND_URL` to comma-separated origins if needed. |
| 503 on health | DB still provisioning — wait 2 min and retry |
| Login works locally but not prod | Re-register or run seed; old JWT from dev won't work |
| AI features fail | Add `GEMINI_API_KEY` in Render env and redeploy API |
